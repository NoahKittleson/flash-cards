• JavaScript (JS)

    • Is a scripting language that is not related to Java. You can use it on webpages and stuff.

  • Operators

    • Parts of the syntax that do something, like +,-,=,/, *. Stuff like that.

  • Variables

    • A designated thing that you assign a value that the computer keeps track of.

  • Variable naming conventions

    • Variable names can't start with numbers, underscores, or include spaces at all. Typical convention uses camelCase.

  • Functions

    • A defined routine that you can give variables and it produces some output (or not).

  • Methods

    • functions that act on an object and that have to be called from the object.

  • Arguments

    • That actual values or variables that you pass into a function.

  • Parameters

    • The generalized values that you list as funciton input.

  • Return

    • The output for a function.

  • Chaining methods

    • When you have multiple calls, usually chaining off of return values. Example: number.asString().asNumber();

  • Strings

    • Characters encased in quotes. Text data.

  • Booleans

    • True or False.

  • Undefined

    • When you haven't assigned anything to a variable, OR when your function has no output it will give UNDEFINED.

  • NaN

    • Not a Number. Occurs in bad parses and dividing by zero errors.

  • Escape

    • \ character, it means that you're no longer taking the literal value of the character it precedes.

  • Alert

    • a popup dialog box

  • Comments

    • Text in a program that has no actual meaning when ran, purely for instructional purposes for programmers.

  • jQuery

    • A library for JavaScript

  • Attributes

    • Anything that you declare inside of the angle brackets, that modifies the tag.
